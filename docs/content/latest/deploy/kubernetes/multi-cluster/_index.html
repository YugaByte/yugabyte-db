--- title: Multi-cluster Kubernetes linkTitle: Multi-cluster description: Deploy on Multi-cluster Kubernetes headcontent: Deploy YugabyteDB natively on multiple Kubernetes clusters. image: /images/section_icons/deploy/kubernetes.png aliases: - /latest/deploy/kubernetes/multi-cluster menu: latest: identifier: deploy-kubernetes-mc parent: deploy-kubernetes weight: 623 --- As highlighted in Kubernetes documentation, a single Kubernetes cluster can run inside a single region only. This means workloads will be distributed across multiple nodes of a single Kubernetes cluster running in a single region. When geo-distributed SQL workloads like YugabyteDB need to run on Kubernetes, the preferred approach is to create a single YugabyteDB cluster that spans multiple geo-distributed Kubernetes clusters.
